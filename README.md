Telegram-бот для розыгрыша **3× MacBook Neo + 1× AirPods 3 Pro**.

## 🚀 Полное руководство по первому запуску

Последовательность действий для запуска бота с нуля на новом сервере:

### 1. Скачивание и установка среды

Для проектов на Python мы используем современный пакетный менеджер `uv`.

```bash
# Клонируем проект
git clone <repo-url>
cd bot_neo

# Создаем виртуальное окружение и устанавливаем все зависимости из pyproject.toml
uv venv --python 3.13
uv sync
```

### 2. Настройка конфигурационного файла

Боту требуются учетные данные для работы с Telegram API.

```bash
cp .env.example .env
```

Отредактируйте файл `.env`:
- `BOT_TOKEN` — вставьте токен вашего бота, полученный от [@BotFather](https://t.me/BotFather).
- `OWNER_IDS` — вставьте цифровые Telegram ID owner-ов через запятую. Owner управляет временными админами, мини-игрой и системными настройками.

### 3. Предварительное наполнение базы данных

Прежде чем бот начнет принимать обычных пользователей, в базу данных (которая будет автоматически создана при первом запуске) необходимо добавить спонсоров и "пасхалки":

```bash
# 1. Добавить каналы, на которые участники должны будут подписаться
uv run python scripts/add_channels.py

# 2. Добавить секретные промокоды ("пасхалки")
uv run python scripts/add_promocodes.py
```

### 4. Запуск бота и Инициализация конкурса

Теперь можно запустить самого бота:

```bash
uv run python main.py
```

> [!WARNING]
> **Важно:** По умолчанию, бот блокирует все команды от обычных пользователей, если не задана дата окончания конкурса!

Как только бот запущен, **администратор должен написать боту в Telegram** команду активации конкурса. 
Напишите:
`/set_date YYYY-MM-DD HH:MM` (например: `/set_date 2026-12-31 23:59`)

После этого конкурс начнётся, и бот будет готов принимать пользователей в штатном режиме!

## 🧪 Локальный запуск Telegram Web App через Pinggy

Мини-апка открывается в Telegram по HTTPS URL из `WEBAPP_URL`, а локально Vite проксирует `/api/*` на Python API бота (`127.0.0.1:8080`). Поэтому наружу нужно пробрасывать порт `5173`, не `8080`.

Запустите три процесса в отдельных терминалах:

```bash
npm run webapp:dev
uv run python main.py
npm run tunnel:pinggy
```

Pinggy каждый раз выдает новый URL. После запуска туннеля скопируйте текущий HTTPS URL и обновите `.env`:

```bash
npm run webapp:url -- https://<actual-pinggy-url>
```

Эту команду можно запускать как из корня репозитория, так и из `plagins/cherry-charm`.

После смены `WEBAPP_URL` обязательно полностью перезапустите `uv run python main.py`: `config.py` читает `.env` только при старте процесса. В Telegram отправьте `/start` заново или выполните `/refresh_menu`, потому что старые кнопки могут содержать прежний URL.

Если в логах старта бот всё равно показывает старый URL, проверьте shell environment:

```bash
env | grep '^WEBAPP_URL='
```

В этом проекте `.env` принудительно имеет приоритет над экспортированными переменными, чтобы stale Pinggy URL не переживал перезапуск терминала.

Для проверки текущей связки:

```bash
npm run webapp:url -- https://<actual-pinggy-url> --check
```

Ожидаемые результаты:

- `local Vite`: `200`
- `local API auth gate`: `401` без Telegram `initData`, это нормально
- `public WebApp`: `200`
- `public API auth gate`: `401` без Telegram `initData`, это нормально

Если Telegram показывает `no tunnel here`, открыт старый Pinggy URL или SSH-сессия туннеля уже закрыта.

Опционально можно запускать Pinggy с локальным debugger API и брать URL автоматически:

```bash
npm run tunnel:pinggy:debug
npm run webapp:url -- --from-pinggy-debugger --check
```

## 🛠 Все функции Администратора

Так как бот предназначен для высокой нагрузки и требует своевременного контроля, в нём реализованы гибкие инструменты администрирования. Owner-ы задаются через `OWNER_IDS`, а временные админы добавляются owner-ом прямо в Telegram и хранятся в БД.

### Telegram-команды

- `/admin_menu`
  **Описание:** Показывает админ-панель. Owner видит owner-разделы и операционные разделы, временный админ видит только операционные инструменты.

- `/owner_menu`
  **Описание:** Owner-only меню для управления доступами, мини-играми и системными настройками.

- `/add_admin <telegram_id>` / `/remove_admin <telegram_id>` / `/list_admins`
  **Описание:** Owner-only управление временными админами.

- `/add_channel <channel_id> | <title> | <invite_link>` / `/remove_channel <channel_id>` / `/list_channels`
  **Описание:** Управление каналами обязательной подписки для owner/temp admin.

- `/add_promocode <code>` / `/add_promocodes <code1> <code2> ...`
  **Описание:** Создание одного или нескольких промокодов для owner/temp admin.

- `/list_plugins` / `/set_plugin <plugin_key>`
  **Описание:** Просмотр доступных мини-игр и owner-only выбор активной мини-игры.

- `/reset_contest` / `/reset_history`
  **Описание:** Owner-only сброс конкурса перед новым раундом. Перед очисткой сохраняет archive snapshot, затем сбрасывает tickets, winners, casino spins, channels, promocodes и temporary admins. `/reset_history` показывает последние reset runs.

- `/set_date YYYY-MM-DD HH:MM`
  **Описание:** Задает или продлевает дату окончания конкурса "на лету" прямо в базе данных, не требуя перезапуска.
  *Примечание: Если конкурс был завершен, использование этой команды перезапустит его и очистит список старых победителей.*
  
- `/admin_stats`
  **Описание:** Полная консолидированная статистика по текущему конкурсу в реальном времени. Показывает:
  * Общее количество регистраций
  * Количество активных допущенных участников
  * Суммарную массу выданных билетов
  * Статистику рефералов и промокодов
  * Число пользователей, отписавшихся/заблокировавших бота

- `/draw`
  **Описание:** Ручной запуск алгоритма случайного выбора победителей на основе их билетов. Перед выбором бот live-проверяет подписку кандидатов на текущие каналы. Запускать **только** когда конкурс был окончен (время прошло). Бот автоматически распределит 4 приза (3 MacBooks + 1 AirPods) и разошлет уведомления.

- `/casino_stats`
  **Описание:** Статистика модуля «🎰 Казик»: спины за сегодня/всего, суммы ставок и выплат, house profit, winrate, разбивка результатов и топ-3 игроков по количеству спинов.

- `/refresh_menu`
  **Описание:** Принудительно обновляет reply-меню у всех пользователей, чтобы сразу применились новые кнопки (например, «🎰 Запустить кампанию»).

- `/webapp_url`
  **Описание:** Показывает текущий `WEBAPP_URL` из запущенного процесса и переустанавливает Telegram WebApp menu button для default menu и текущего чата. Полезно после смены локального Pinggy URL.

- `/send <текст>`
  **Описание:** Админ-рассылка по всем пользователям. Сначала показывает предпросмотр и кнопки подтверждения/отмены, после подтверждения отправляет сообщение всем записям из `users`.

### Терминальные скрипты (CLI)

- `uv run python scripts/add_channels.py` — Интерактивный терминальный интерфейс для добавления каналов спонсоров. Запросит ID канала, его название и ссылку на вступление.
- `uv run python scripts/add_promocodes.py` — Генератор/загрузчик скрытых пасхалок (промокодов) в БД, которые пользователи смогут находить у вас в соц-сетях.
- `uv run python scripts/diagnose_draw.py` — Аудит здоровья розыгрыша. Запуск скрипта анализирует готовность базы данных к `/draw` (хватает ли участников и настроены ли все ключи).

## 🎰 Casino Module

Модуль «Казик» добавляет отдельную игровую механику поверх билетов участника.

### Логика выигрыша (Telegram Dice 🎰)

Telegram возвращает число `1..64`, которое декодируется в 3 символа барабанов. Выигрыш засчитывается только за три одинаковых символа:

- `7-7-7` (value `64`) → `jackpot`, множитель `x4`, чистый профит `+bet*3`
- `BAR-BAR-BAR` (value `1`) → `win`, множитель `x2`, чистый профит `+bet*1`
- `🍇-🍇-🍇` (value `22`) → `win`, множитель `x2`, чистый профит `+bet*1`
- `🍋-🍋-🍋` (value `43`) → `win`, множитель `x2`, чистый профит `+bet*1`
- Любая другая комбинация (например `7-7-🍋`, value `48`) → `loss`, множитель `0`, чистый профит `-bet`

Формула баланса после спина:

`new_balance = old_balance - bet + (bet * multiplier)`

### Ограничения и безопасность

- Минимальный баланс для входа: `2` билета.
- Допустимые ставки: `1..5`.
- Ставка доступна только если после выбора остаётся минимум `1` билет (`balance - bet >= 1`).
- Лимит: `3` спина в календарные сутки `Europe/Kiev`.
- Все операции баланса выполняются строго в транзакции `BEGIN IMMEDIATE ... COMMIT/ROLLBACK`.
- Повторные клики во время анимации блокируются в single-process памяти (`spinning` lock).
- Все игровые события и ошибки пишутся в `logs/casino.log`.

### Данные и таблицы

Добавлены таблицы:

- `casino_spins` — журнал спинов (ставка, dice_value, multiplier, баланс до/после, результат).
- `user_flags` — one-time флаги пользователя (включая подтверждение дисклеймера).

Индекс для дневных лимитов:

- `idx_casino_daily (user_id, created_at)`
